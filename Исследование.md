# Подробная выжимка по исследованию: приложение для виз и туров

**Главный вывод:** приложение нужно строить не как “каталог виз и туров”, а как **систему диагностики и сборки поездки**.

Формула продукта:

```text
Пользователь выбирает свою ситуацию
→ проходит диагностику поездки
→ получает статус готовности / риски / пробелы
→ приложение предлагает нужный путь:
   виза / страховка / тур / трансфер / сопровождение / первые 72 часа
→ заявка / оплата / консультация / партнёрский сервис
```

Это сильнее, чем просто:

```text
Визы → Туры → Страховка
```

Потому что пользователь платит не за “список услуг”, а за **снятие неопределённости**.

---

# 1. Короткий стратегический диагноз

## Что ты реально строишь

Ты строишь не просто приложение для виз и туров. Сильнее назвать это так:

> **Travel Readiness App** — приложение, которое помогает человеку понять, готова ли его поездка, что ему нужно оформить, какие риски есть и какой тур / сервис подходит под его ситуацию.

Для Санкт-Петербурга и иностранцев это можно упаковать как:

> **SPB Entry OS** — личный пульт въезда, визы, страховки, маршрута и первых 72 часов в городе.

Для граждан РФ, которые едут за границу:

> **VisaTrip Control Tower** — приложение, которое ведёт человека от идеи поездки до визы, страховки, тура и сопровождения.

# 3. Главная продуктовая идея

## Не так

```text
Приложение для виз и туров
```

Это звучит как обычный агрегатор.

## Лучше

```text
Приложение, которое показывает человеку правильный путь поездки:
нужна ли виза, что не готово, какие риски есть,
какую страховку оформить, какой тур выбрать и нужна ли поддержка.
```

## Самая сильная формула

```text
Виза + страховка + тур + готовность поездки + сопровождение
```

Именно **готовность поездки** должна быть центральной идеей.

# 4. Правильная логика первого экрана

Твой вариант:

```text
1) Гражданин РФ → узнать шанс получения визы → виза + страховка → туры
2) Гражданин РФ, виза есть → туры
3) Иностранец → ...
```

Рабочий, но его надо усилить.

## Главная проблема

Фраза **“узнать шанс получения визы”** опасная. Она звучит как обещание вероятности. Лучше заменить на:

* **Проверить визовую готовность**
* **Подобрать визовый путь**
* **Понять, что нужно для визы**
* **Оценить готовность документов**
* **Проверить поездку перед подачей**

# 6. Три основных пользовательских пути

## Путь 1 — Гражданин РФ, визы нет

```text
Гражданин РФ
↓
Выбор страны
↓
Цель поездки: туризм / бизнес / лечение / семья / учёба
↓
Даты поездки
↓
Проверка визового сценария
↓
Visa Readiness Score
↓
Документы
↓
Страховка
↓
Подача через партнёра / консультация
↓
Тур / маршрут
↓
Дополнительные услуги
```

### Что продавать

| Продукт                  | Почему купит                    |
| ------------------------ | ------------------------------- |
| Визовая консультация     | Боится ошибки                   |
| Проверка документов      | Не хочет отказа из-за мелочи    |
| Страховка                | Нужна для поездки / визы        |
| Тур                      | Уже есть намерение ехать        |
| Пакет “поездка под ключ” | Хочет закрыть всё в одном месте |

### Живой пример

Пользователь хочет в Италию / Испанию / Францию. Он не понимает, какие документы нужны, сколько денег показать, какую страховку купить, какие сроки записи. Приложение не говорит “шанс 87%”, а показывает:

```text
Готовность к визе: 46/100

Не хватает:
— подтверждения маршрута
— страховки
— финансовых документов
— брони проживания
— проверки срока паспорта

Рекомендуемый следующий шаг:
Проверка визового пакета + страховка
```

---

## Путь 2 — Гражданин РФ, виза уже есть

```text
Гражданин РФ
↓
Виза уже есть
↓
Выбор страны / города
↓
Проверка срока визы, количества въездов и дней пребывания
↓
Страховка
↓
Тур / отель / маршрут
↓
Дополнительные услуги
```

### Важная механика

Даже если виза есть, пользователь может ошибиться:

* срок визы заканчивается;
* недостаточно дней пребывания;
* виза однократная, а маршрут с несколькими въездами;
* нет страховки;
* паспорт скоро заканчивается;
* ребёнку нужны отдельные документы;
* страна изменила правила.

Поэтому лучше не сразу показывать туры, а сделать мини-проверку:

```text
Проверить, всё ли готово к поездке
```

### Что продавать

| Продукт             | Почему купит            |
| ------------------- | ----------------------- |
| Тур                 | Основная потребность    |
| Страховка           | Обязательная / полезная |
| Отель               | Логичная допродажа      |
| Трансфер            | Удобство                |
| Экскурсии           | Дополнительный чек      |
| Проверка готовности | Снятие риска            |

Иностранец хочет приехать в Петербург на 5 дней. Приложение спрашивает:

```text

```

```text

— entry path not confirmed
— insurance not selected

— no arrival transfer
— no first 72h plan

Entry Check + First 72h in Saint Petersburg
```

# 7. Главное ядро приложения

## Центральная фича №1 — Trip Readiness Score

Это сердце продукта.

Пользователь должен получить не просто список услуг, а **оценку готовности поездки**.

```text
Готовность поездки: 58/100
```

## Что оценивает score

| Блок            | Что проверяется                    | Как монетизируется            |
| --------------- | ---------------------------------- | ----------------------------- |
| Виза            | нужна / не нужна / есть / неясно   | визовая консультация          |
| Документы       | паспорт, фото, анкета, сроки       | проверка документов           |
| Страховка       | нужна / есть / не выбрана          | продажа страховки             |
| Деньги / оплата | способ оплаты, валюта, ограничения | консультация / предупреждение |
| Перелёт         | даты, аэропорт, стыковки           | тур / пакет                   |
| Проживание      | отель, регистрация, адрес          | отель / партнёр               |
| Трансфер        | как добраться                      | маржа / комиссия              |
| Тур             | цель поездки, интересы             | продажа тура                  |
| Поддержка       | переводчик, concierge, emergency   | премиум-пакет                 |

# 9. Продуктовая архитектура

## Уровень 1 — Бесплатный вход

**Цель:** получить лид.

Что внутри:

* выбор ситуации;
* базовая диагностика;
* readiness score;
* общий чеклист;
* рекомендации;
* предложение платного шага.

---

## Уровень 2 — Платная проверка

**Название:** `Visa / Entry Readiness Check`

Что внутри:

* персональный разбор;
* проверка документов;
* чеклист;
* предупреждения;
* рекомендации;
* PDF / личный план.

Пример цены:

```text
990–4990 ₽ для РФ-направлений
49–149 € для иностранцев
```

---

## Уровень 3 — Основной коммерческий пакет

**Для граждан РФ:**

```text
Виза + страховка + тур
```

**Для иностранцев:**

```text
Entry Check + First 72h in SPB
```

Что входит:

* проверка;
* страховка;
* тур / маршрут;
* трансфер;
* поддержка;
* партнёрские услуги.

---

## Уровень 4 — Премиум

**Название:** `Concierge Trip`

Что входит:

* персональный менеджер;
* водитель;
* переводчик;
* private route;
* рестораны;
* мероприятия;
* помощь до и во время поездки.

Пример цены:

```text
от 700 € до 3000 €+
```

| Карта достопримечательностей     | Не даёт уникальности                 |
| Полное бронирование отелей       | Тяжёлая интеграция                   |
| Социальная сеть путешественников | Не связано с первой монетизацией     |
| Нативные приложения iOS/Android  | Имеет смысл после доказанной воронки |

# 16. Главная бизнес-логика

## Не продавать пользователю всё сразу

Сначала надо понять его состояние.

```text
Не знает, нужна ли виза → диагностика
Нужна виза → проверка документов
Виза есть → тур
Иностранец → entry path
Бизнес → concierge
Семья → безопасный пакет
```

## Приложение должно быть воронкой

```text
Бесплатная диагностика
↓
Платная проверка
↓
Страховка / партнёрская услуга
↓
Тур
↓
Сопровождение / premium
```

# 17. Главные риски

| Риск                                   | Почему опасен                                      | Как снизить                                    |
| -------------------------------------- | -------------------------------------------------- | ---------------------------------------------- |
| Неверные визовые данные                | Может навредить пользователю                       | Только официальные источники + дата обновления |
| Обещание “шанса визы”                  | Может выглядеть как гарантия                       | Использовать “готовность”                      |
| Продажа турпакета без юридической базы | Возможны требования к туроператорской деятельности | Юрист + партнёрская модель                     |
| Слишком много стран                    | Невозможно поддерживать правила                    | Старт с 5–10 направлений                       |
| Слишком широкий продукт                | Пользователь не поймёт ценность                    | Центральный сценарий: готовность поездки       |
| AI ошибается                           | Визы требуют точности                              | Human review                                   |
| Партнёры портят качество               | Бьёт по бренду                                     | Проверенная база исполнителей                  |
| Нет платного next step                 | Приложение станет справочником                     | После score всегда оффер                       |

# 18. Приоритет стран / направлений

## Для граждан РФ

Начинать лучше не со всех стран, а с тех, где:

* высокий спрос;
* понятная визовая боль;
* можно продавать туры;
* есть партнёры;
* можно быстро обновлять правила.

Пример стартовой логики:

| Тип направления | Почему интересно                                      |
| --------------- | ----------------------------------------------------- |
| Шенген          | Высокая боль, сложные правила, высокий спрос          |
| ОАЭ             | Часто туристическое направление, можно продавать туры |
| Турция          | Массовый туризм, страховки, маршруты                  |
| Китай           | Растущий интерес                                      |
| страны Азии     | Комбинация виз / туров / страховок                    |

## Для иностранцев в РФ / СПб

Стартовать с тех рынков, где уже есть поток в Петербург: Китай, Иран, Турция, Саудовская Аравия, Индия — именно эти страны указаны в топе дальнего зарубежья по данным Петербурга за 2025 год. ([gov.spb.ru][3])

# 23. Финальная рекомендация

## Строить приложение так:

```text
Первый экран:
“С чего начинается ваша поездка?”

1. Мне нужна виза
→ проверка визовой готовности
→ документы
→ страховка
→ тур

2. Виза уже есть
→ проверка готовности поездки
→ тур
→ страховка
→ трансфер / отель / экскурсии

3. I want to visit Russia / Saint Petersburg
→ check entry path
→ e-visa / visa-free / regular visa
→ insurance
→ first 72h in SPB
→ tours / transfer / concierge
```

## Главная фича

```text
Trip Readiness Score
```

## Главный платный продукт

```text
Visa / Entry Readiness Check
```

## Главный upsell

```text
Страховка + тур
```

## Главный премиум-продукт

```text
First 72h / Concierge
```

## Главная ошибка

Сделать приложение как обычный агрегатор:

```text
Визы
Туры
Страховки
Отели
Блог
```

## Сильная версия

Сделать приложение как **путь решения**:

```text
Я хочу поехать
→ я не понимаю, что нужно
→ приложение показывает мой путь
→ показывает риски
→ предлагает платный следующий шаг
→ закрывает визу, страховку, тур и поддержку
```

Это уже не просто “приложение для виз и туров”.
Это **система подготовки поездки**, где визы и туры становятся частью одной управляемой логики.
