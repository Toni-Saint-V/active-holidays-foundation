# Figma Make Prompt: Visa Readiness Pass

Create one premium mobile-first screen concept for an AI-powered visa/travel readiness app.

Screen name: Visa Readiness Pass.

Viewport: iPhone 14, 390 x 844.

Style:

- dark premium
- travel-tech 2026
- restrained contrast
- typography-led hierarchy
- compact, calm, serious
- no colorful consumer-game look
- no decorative map clutter
- no generic AI glow
- no marketing landing page

Use exactly five components:

1. Header + Trip Summary
2. Visa Readiness Status
3. Interactive Route Risk Map
4. Documents Impact Panel
5. Alternatives + Primary CTA

Maximum three visible actions.

Do not show visa outcome likelihood.
Do not imply legal certainty.
Do not let AI decide for the user.
Do not invent API routes, verdict states, or result models.

Component details:

1. Header + Trip Summary
   - route label
   - pass title
   - last updated label

2. Visa Readiness Status
   - canonical status label
   - document readiness label
   - evidence label
   - short why line

3. Interactive Route Risk Map
   - 3-5 route points
   - selected point state
   - each point shows status, risk, requirements
   - use restrained highlight or heat indicator

4. Documents Impact Panel
   - ready/required count
   - document readiness completion
   - missing or blocked documents
   - impact on next step

5. Alternatives + Primary CTA
   - one primary next action
   - up to two secondary actions
   - alternatives are compare-only

AI boundary:

- AI may explain missing data, rule-based logic, risks, and next step.
- Every AI-assisted explanation must include a short why.
- AI must not forecast visa outcome or make the decision.

Output only one screen concept.
Do not generate implementation code.
Do not add extra screens.
