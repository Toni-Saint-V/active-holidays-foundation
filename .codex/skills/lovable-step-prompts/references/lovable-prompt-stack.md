# Lovable Prompt Stack

Use this order inside each Lovable prompt.

## 1. Mission

- Что за экран или flow
- Зачем он существует
- Что пользователь должен понять или сделать

## 2. Scope

- Какие route(s) строим
- Какие блоки обязательны
- Какие состояния обязательны

## 3. Contracts

- payload keys
- route names
- CTA behavior
- what is presentational vs domain-bound

## 4. Visual direction

- tone
- references by mood
- color / motion constraints
- mobile-first expectations

## 5. Non-negotiables

- no product invention
- no fake entities
- no dark patterns
- no hidden states
- preserve Russian copy if the product is Russian

## 6. Finish line

- what counts as done
- what should be easy for Codex to wire afterwards

## Practical Lovable tips

- Better to give one sharp prompt per step than one giant prompt for everything.
- Separate structure prompts from polish prompts when the UI is still unstable.
- When an existing screen exists, say what must stay and what can change.
