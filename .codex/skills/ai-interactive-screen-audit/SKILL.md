# AI Interactive Screen Audit

## Goal

Ensure every important screen is not only usable, but intelligently helpful and interaction-rich in a product-valid way.

## Quick start

- Write all user-facing output in Russian.
- Load `references/ai-interactivity-checklist.md`.
- Audit the actual screen goal first, then decide whether AI belongs there at all.

## Workflow

1. Understand the screen job.
   - What is the user trying to decide, understand, or complete?
   - Where is uncertainty, friction, or waiting?
   - Is there a trust moment that a smart layer could improve?

2. Review AI-native opportunities.
   - Useful AI patterns include:
     - live preview
     - explainability
     - smart defaults
     - ambiguity highlighting
     - what-if simulation
     - recovery guidance
     - human handoff packaging
   - Reject AI where it only adds noise or fake autonomy.

3. Review interactivity opportunities.
   - Useful interactive patterns include:
     - progressive disclosure
     - compare modes
     - sliders and scenario testing
     - swipe / sort / filter that changes understanding
     - direct manipulation of meaningful state

4. Prioritize upgrades.
   - Split findings into:
     - must add
     - strong upgrade
     - optional delight
   - Prefer low-complexity, high-signal upgrades first.

## Output

- `Что экран уже делает`
- `Где здесь нужен AI`
- `Какие интерактивы реально усилят экран`
- `Что не стоит добавлять`
- `Приоритетный upgrade list`

## Rules

- Do not add AI where deterministic UI is better.
- Do not add motion or interaction that does not improve comprehension or control.
- Do not confuse novelty with product strength.
