# AH UI Direction

## Goal

Define a strong UI direction before implementation.

## When To Use

- UI critique or redline
- PNG preview / design approval
- layout hierarchy, CTA, spacing, mobile-first decisions
- Russian product copy in UI
- visual direction before Lovable/Cursor/Codex implementation

## Workflow

1. Respect the PNG approval gate before UI code.
2. Judge first viewport clarity, hierarchy, CTA, and trust.
3. Keep visual language restrained and product-specific.
4. Write concise Russian UI copy.
5. Use internal references only when needed:
   - `_internal/frontend-premium-ui.md`
   - `_internal/design-system-enforcer.md`
   - `_internal/product-ux-flow-review.md`
   - `_internal/a11y-trust-usability.md`
   - `_internal/russian-trust-safe-copy.md`
   - `_shared/active-holidays/premium-ui-playbook.md`

## Hard Rules

- No UI implementation before approved PNG.
- No generic SaaS card soup.
- No decorative complexity that weakens the decision flow.
