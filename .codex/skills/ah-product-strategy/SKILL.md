# AH Product Strategy

## Goal

Turn product ambiguity into one strong executable next step.

## When To Use

- пользователь спрашивает "что делать дальше"
- нужно подготовить Notion / Lovable / Cursor / Codex handoff
- нужно оценить продуктовый план, prompt, scope или phase readiness
- задача больше про ценность, поток и доверие, чем про код

## Workflow

1. Identify the current phase and source of truth.
2. State the product goal, user goal, and strongest next task.
3. Keep the output paste-ready when it goes to another tool.
4. Avoid generic brainstorming; choose a high-leverage direction.
5. Use internal references only when needed:
   - `_internal/product-ux-flow-review.md`
   - `_internal/docs-and-handoff.md`
   - `_shared/active-holidays/product-context.md`

## Hard Rules

- Do not invent user stories, labels, metrics, or source-of-truth facts.
- Do not turn product tasks into technical busywork.
- Keep next steps concrete and executable.
